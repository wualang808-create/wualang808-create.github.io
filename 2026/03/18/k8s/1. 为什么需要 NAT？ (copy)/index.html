1. 为什么需要 NAT？ (copy)

发表于2026-03-18|更新于2026-04-09|k8s
|浏览量:

NAT = Network Address Translation 网络地址转换

SNAT = source 。。。

DNAT = destination NAT

1. 为什么需要 NAT？

  • IPv4 地址枯竭：全球的公网 IPv4 地址只有约 43 亿个。如果没有 NAT，每个手机、电脑、智能灯泡都要占一个公网 IP，地址早就用光了。
  • 安全保护：NAT 隐藏了内网设备的真实 IP。外部黑客只能看到路由器的公网 IP，无法直接发起攻击到你的电脑。

2. NAT 是如何工作的？（以你家路由器为例）

假设你的电脑私有 IP 是 192.168.1.5，你想访问百度（公网 IP 110.242.68.3）：

  1. 发出请求：你的电脑把数据包发给路由器。
  2. 地址转换：路由器把数据包里的“源地址”从 192.168.1.5 换成路由器的公网 IP（比如 202.10.1.1），并记录在一个 NAT 映射表里。
  3. 外部响应：百度把数据发回到路由器的公网 IP。
  4. 分发回传：路由器查表发现这个请求是 192.168.1.5 发起的，于是把数据包转发给你的电脑。
类型 别名 特点 常见场景
静态 NAT 一对一 一个私网 IP 永久对应一个公网 IP。 企业内部服务器对外发布。
动态 NAT 多对多 一组私网 IP 轮流使用一组公网 IP。 较少见，因公网 IP 资源依然紧张。
PAT (NAPT) 多对一 最常用。 多个私网 IP 共享一个公网 IP，通过不同的端口号来区分。 家庭 WiFi 路由器。

4. NAT 的优缺点

  • 优点：
    • 极大节省了公网 IP 资源。
    • 增加了内网隐蔽性和安全性。
    • 方便管理，更换运营商时不需要更改内网配置。
  • 缺点：
    • 性能损耗：路由器需要拆包、改地址、重新计算校验和。
    • 破坏端到端通信：某些应用（如 P2P 下载、老式 VoIP、部分联机游戏）在 NAT 后面会遇到连接困难。

5. 与 iptables 的联系

在你之前提到的 iptables 中，NAT 是通过 nat 表来实现的：

  • SNAT (Source NAT)：修改源地址，通常用于内网访问外网。
  • DNAT (Destination NAT)：修改目的地址（端口转发），通常用于外网访问内网服务器。
  • MASQUERADE（伪装）：动态 SNAT，适用于拨号上网等公网 IP 不固定的情况。
