🚀 Hexo 隐私化与自动化部署终极指南

  3. 私有仓库与 SSH 安全对接（隐私化）
  4. GitHub Actions 自动化配置（云端大脑）
  5. Python 本地自动化脚本（一键发布）
  6. 桌面效率化封装（傻瓜式操作）

第一部分：环境配置

在操作之前，必须确保电脑安装了以下“三大件”：

  1. Node.js: 下载 LTS 版本。它是 Hexo 运行的引擎。
  2. Git: 下载 Git。负责把你的代码传到 GitHub。
  3. Python 3: 下载 Python。重要： 安装时务必勾选 Add Python to PATH。

第二部分：Hexo 初始化与主题

我们要建立博客的本地源码目录。

  1. 新建文件夹：在 D 盘新建 Myblog 文件夹。

  2. 初始化：在 D:\Myblog 内右键点击 Git Bash Here，输入：

    Bash

  3. 安装 Butterfly 主题：

    Bash

    • 修改配置：打开根目录 _config.yml，找到 theme: 改为 butterfly。

  4. 安装搜索插件（后续自动化必须）：

    Bash

  5. 开启本地搜索：

    • 站点配置文件 (_config.yml) 底端加入：

      YAML

    • 主题配置文件 (themes/butterfly/_config.yml) 找到 local_search：

      YAML

第三部分：私有仓库与 SSH 对接

为了保护源码不被公开（比如你的草稿、插件配置等），我们需要把源码放在私有库。

  1. 创建私有仓库：在 GitHub 上创建一个名为 blog-main 的仓库，选 Private。

  2. 生成 SSH 密钥：在 Git Bash 输入：

    Bash

  3. 绑定 GitHub：

    • 找到 C:\Users\你的用户名\.ssh\id_rsa.pub，用记事本打开并复制内容。
    • 去 GitHub 个人设置 -> SSH and GPG keys -> New SSH Key，粘贴进去。

  4. 本地仓库关联：

    Bash

第四部分：GitHub Actions 云端配置

让 GitHub 服务器帮你跑 hexo generate。

1. 设置 Secret（保险箱）

  • 打开 GitHub 仓库 blog-main -> Settings -> Secrets and variables -> Actions -> New repository secret。
  • Name: DEPLOY_KEY
  • Value: 粘贴你电脑上的 id_rsa（私钥，不是带 .pub 的那个）的全部内容。

2. 编写 workflow

在博客根目录创建文件夹结构：.github/workflows/，在里面新建文件 deploy.yml：

YAML

第五部分：Python 本地全自动脚本

这是最核心的部分，解决文章格式、缓存、同步冲突等问题。

在根目录新建 auto_sync.py：

Python

第六部分：桌面效率化（最后封装）

  1. 快捷文章入口： 右键 D:\Myblog\source\_posts -> 发送到桌面快捷方式。改名为 “1. 稿件投递箱”。
  2. 一键发布按钮： 桌面右键 -> 新建快捷方式 -> 填入： cmd /k python D:\Myblog\auto_sync.py 改名为 “2. 一键发布博客”。

📖 总结：如何复现成功？

  • 新博主操作：只需要把任何 .md 文件（哪怕没有标题和日期）丢进 “稿件投递箱”，然后双击 “一键发布博客”。
  • 底层逻辑：Python 脚本补全格式 -> 清理缓存 -> 推送源码 -> GitHub Actions 识别推送 -> 在云端用 20 核心的服务器帮你编译生成 HTML -> 自动更新到网页。
  • 遇到未更新怎么办：网页上按下 Ctrl + Shift + R。
