设置key-value

Redis

  • 关系型数据库 MySQL =》类似Excel （行-记录、列-字段）

  • 内存型数据库 Redis =》数据存储在内存（快）/key-value

  • 文档型数据库 MongeDb

写开发项目-》需求分析–>技术选型（编程/web框架/数据库）

  • 功能

    • （分布式）缓存 ，挡在mysql 前面

    • redis 是内存型数据库–》 支持数据持久化（磁盘）

    • 支持高可用（有机器宕机，业务也能正常提供服务-解决单点问题）

    • 数据存储key-value ，支持队列（消息对列），支持排序

    • 性能：读速度11W/s ，写8.1W/s

    • 数据类型丰富：

  • 安装

    • Downloads | Redis

    • 版本：大版本中 偶数是稳定版 奇数是测试版

    • 退出

      • CTRL+D
      • exit

    • 关掉服务

      • 在命令行执行 shutdown

  • 赋值、获取、删除

  • set。。。get

  • 过期时间设置（创建数据时）

    • ex：设置过期秒数

    • 验证码登录：

  • 时间戳：1970-1-1至今秒数

  • data

  • 继承过期时间

=======================================
Redis常用命令
set k1 1 => k1(key),1(value)

  • 命令不区分大小写，key区分大小写
  • 帮助：help @类型

设置key-value

Set
name => 字符串 => 数字、字母、:、-
127.0.0.1:6379> SET k2 100
OK

获取数据

127.0.0.1:6379> get k2
“100”
127.0.0.1:6379> get K2
(nil)

查看帮助

127.0.0.1:6379> help @string

APPEND key value
summary: Appends a string to the value of a key. Creates the key if it doesn’t exist.
since: 2.0.0

查看当前库所有的Key

127.0.0.1:6379> set a1 “Hello World”
OK
127.0.0.1:6379> keys *

  1. “a1”
  2. “k2”
  3. “k1”
    127.0.0.1:6379> keys k*
  4. “k2”
  5. “k1”

判断Key是否存在 => 有几个存在就返回几

127.0.0.1:6379> exists k1
(integer) 1

127.0.0.1:6379> exists k1 k2 a1 a2 a5
(integer) 4

查看key的类型

127.0.0.1:6379> type k1
string

删除指定key的数据 (阻塞删除)

127.0.0.1:6379> get k1
“1”
127.0.0.1:6379> del k1
(integer) 1
127.0.0.1:6379> get k1
(nil)

unlink: 非阻塞删除 => 仅仅将key从keyspace元数据中删除，真正删除会在后续异步完成（大量数据删除）

127.0.0.1:6379> get k2
“100”
127.0.0.1:6379> unlink k2
(integer) 1

ttl: 查看还有多少秒过期, -1永不过期,-2已经过期，n秒数

127.0.0.1:6379> ttl k2
(integer) -2
127.0.0.1:6379> ttl a1
(integer) -1
127.0.0.1:6379>

expire: 为key设置过期时间

127.0.0.1:6379> expire a1 60
(integer) 1
127.0.0.1:6379> ttl a1
(integer) 55
127.0.0.1:6379> ttl a1
(integer) -2

select: 切换数据库

Redis中有16个数据库0~15, 默认：0
127.0.0.1:6379> select 1
OK
127.0.0.1:6379[1]> keys *
(empty array)
127.0.0.1:6379[1]> set m1 abc
OK
127.0.0.1:6379[1]> keys *

  1. “m1”

move: 移动key到其他数据库

127.0.0.1:6379[1]> move m1 0
(integer) 1
127.0.0.1:6379[1]> keys *
(empty array)
127.0.0.1:6379[1]> select 0
OK
127.0.0.1:6379> keys *

  1. “m1”
  2. “a2”

dbsize: 查看当前库有多少个key

127.0.0.1:6379> dbsize
(integer) 2

flushdb: 清空当前数据库

127.0.0.1:6379[1]> flushdb
OK

flushall: 清空所有数据库

127.0.0.1:6379[1]> flushall
OK

指定区间

getrange key start end：获取指定区间

setrange key offset value :设置指定区间
