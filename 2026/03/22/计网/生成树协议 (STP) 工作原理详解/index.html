  • 交换机工作原理

    • 微型的Linux系统 有cpu有memory
    • 初始状态：不知道外界情况
    • mac地址表(内存中存放)：记录哪些电脑的mac地址连接到交换机的哪些接口
    • 1 学习：学习帧的源mac地址，知道数据的源头，知道交换机的接口连接哪些电脑
    • 2 转发：广播（给连接的每台机器发信息验证） 对应的机器单播过去 如果mac地址表有转发的时候需要的目的mac地址，就单播，如果没有就短播；如果交换机收到一个组播（小范围广播）或者广播包，也广播
      • 组播：mac地址的第八位是1
      • 广播：mac地址全部是一，十六进制的表示是ff:ff:ff:ff
    • 什么时候广播：
      • 不知道目的mac
      • 接收到组播或广播
    • 广播风暴：
      • 大量的广播产生，交换机于交换机相连 ，形成环路 ，导致交换机瘫痪，不能正常转发数据，因为消耗了整个交换机的cpu、内存、（网络带宽资源）
    • 解决：
      • 非网管交换机：检查物理线路，拔掉一根网线线避免环路
      • 网管交换机：STP（生成树）协议（spanning-tree protocol ）检测到时 会自动堵塞其中一个连接口

生成树协议 (STP) 工作原理详解

生成树协议（STP, Spanning Tree Protocol）的核心任务是：在存在物理环路的分组交换网络中，通过逻辑上断开某些端口，构建一棵无环的“树”，从而防止广播风暴。

它的工作过程可以看作是一场“海选”，通过发送 BPDU（桥接协议数据单元） 报文来协商。

1. 核心选举四个步骤

第一步：选举根桥 (Root Bridge)

整个网络（生成树）只能有一个“大脑”，即根桥。

  • 规则：比较网桥 ID（Bridge ID = 优先级 + MAC 地址）。
  • 胜出者：Bridge ID 最小 的交换机成为根桥。
  • 默认情况：优先级默认为 32768，若优先级相同，则 MAC 地址最小 的交换机胜出。

第二步：选举根端口 (Root Port, RP)

在 每一台非根交换机 上，选出一个离根桥“最近”的端口。

  • 判定标准（优先级从高到低）：
    1. 到达根桥的 路径开销 (Cost) 最小。
    2. 直连的上游交换机的 Bridge ID 最小。
    3. 直连的上游交换机的 Port ID 最小。
  • 结果：每个非根交换机有且只有一个 RP。

第三步：选举指定端口 (Designated Port, DP)

在 每一条链路（网段） 上，选出一个负责转发数据的端口。

  • 规则：比较该链路两端交换机到达根桥的开销。
  • 结果：
    • 根桥上的所有端口通常都是指定端口（DP）。
    • 每一条物理链路上 必须且只有一个 DP。

第四步：阻塞剩余端口 (Blocking)

除了上述选出的 RP 和 DP，剩下的所有端口都会被设置为 阻塞状态 (Blocking)。

  • 状态逻辑：这些端口不转发用户数据帧，也不学习 MAC 地址，但会持续 监听 BPDU。
  • 意义：一旦主链路断开，阻塞端口会在一段时间后“激活”，重新计算并恢复连通性。

2. 端口状态的演变过程

STP 为了防止在计算过程中产生临时环路，端口会经历以下状态演变：

状态 数据转发 地址学习 接收 BPDU 说明
Blocking (阻塞) 否 否 是 初始状态，防止环路。
Listening (监听) 否 否 是 确定端口角色（RP/DP）。
Learning (学习) 否 是 是 开始构建 MAC 地址表。
Forwarding (转发) 是 是 是 正常工作状态。
Disabled (禁用) 否 否 否 端口行政性关闭。

3. ⚠️ 避坑指南：STP 的局限性

经典的 STP (802.1D) 有一个致命缺点：收敛速度慢。从链路故障到恢复通常需要 30-50 秒。在现代网络中，这会导致严重的业务中断。

现代改进方案：

  • RSTP (快速生成树协议)：优化了状态切换机制，实现 毫秒级 收敛。
  • MSTP (多生成树协议)：结合 VLAN，实现不同 VLAN 走不同路径，达到 负载均衡。

交换机以太网接口单双工模式（交换机之间要协商用哪个）：

  • 单工：

    • 两数据站之间 ，只能沿一个方向传输数据
      • 类似于麦克风和扬声器

  • 半双工（half duplex）：

    • 两个数据站之间可双向传输数据,但不能同时进行
      • 类似于对讲机

  • 全双工（full duplex）：

    • 两个数据站可双向且同时进行数据传输
      • 类似于打电话

  • ethtool 网卡名 查看

  • mtu（maximum transmission unit）：最大传输单元（数据站之间也要协商）

    • 比如 : 1500 mtu 最打传输字节数为1500

  • MAC地址 不需要配置，是嵌入在网卡的芯片的，自带，默认全球唯一

    • 可临时修改 ，一般不修改

  • MAC地址表里的条目

    • 如果一个接口对应多个mac地址，可以断定这个接口对面接的是一个交换机；因为交换机可连接多台机器，会有很多mac地址
